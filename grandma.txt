Grandma Explanation Of This Project

We made a small hotel business dashboard.

Think of it like a helper notebook for hotel managers. It looks at hotel data,
finds which hotels might be losing money or having problems, and then lets an AI
explain the situation in plain words.

What The App Does

The app reads hotel performance data from this file:

data/hotel_performance.csv

It shows important numbers like:

- Revenue
- Bookings
- Cancellation rate
- Average daily rate gap
- Risky hotels that need attention

It also has a chat box where someone can ask:

"Which hotel needs attention and what should we do next?"

Then the AI gives a business-style answer.

The Main Files

streamlit_app.py

This is the Streamlit version of the app. This is the file Streamlit Cloud uses
when we host the app on streamlit.app.

app/main.py

This is the FastAPI version of the app. It serves the custom HTML page.

app/static/index.html

This is the nicer-looking local page with the green background, big cards, and
custom design. That is why the local version looked prettier than the hosted
Streamlit version.

app/core/settings.py

This file reads settings from environment variables, such as the AI provider and
API key.

app/core/llm.py

This file talks to the AI provider. For Groq, it sends the key as a bearer token
to the Groq chat completions API.

The API Key We Used

We used a Groq API key.

The environment variable name is:

GROQ_API_KEY

The key itself starts with:

gsk_

Important: we did not save the full real key inside this file or inside the
code, because this project is going to GitHub. Putting the real key on GitHub is
like writing a house key on a public notice board.

Instead, the real key should be added in the hosting platform's secret settings.

For Streamlit Cloud, add these secrets:

LLM_PROVIDER = "groq"
GROQ_API_KEY = "paste_the_real_key_here"
GROQ_MODEL = "llama-3.3-70b-versatile"

For local development, the key can go in a private .env file:

LLM_PROVIDER=groq
GROQ_API_KEY=paste_the_real_key_here
GROQ_MODEL=llama-3.3-70b-versatile

The .env file is ignored by Git because we added it to .gitignore.

What We Changed To Make It Safe For GitHub

We added:

.gitignore

This tells Git to ignore private and generated files, including:

- .env
- Python cache folders
- Streamlit secrets
- virtual environments

We added:

.env.example

This is a safe example file. It shows people what settings are needed, but it
does not contain the real API key.

We also removed Python cache files from the Git index so they do not get pushed.
Those files are generated by Python and do not belong in GitHub.
